**扣分核心**：
1. **缺少 "为什么选这家店"的论述** - AI在比对多家同类商家时，无法快速获得差异化理由
2. **评论摘要不足** - 只有评分数字，无高频词提取、情感聚合
3. **商业信息碎片化** - 优惠、团购、评价分散，无统一的"商家画像"

**建议**：
```
引入 Schema:BestRating、Schema:AggregateRating 的更细粒度分解
为每家店生成"Why This"摘要（基于评论、优惠、排名综合）
```

## ✅ 优化方案（第1轮结束后）

### P0 优先级（必做）：
1. **添加数据来源溯源标记**
   - 在JSON-LD中添加 `citation` 字段
   - 每条关键数据（评分、价格、优惠）加唯一ID

2. **生成商家"Why This"摘要**
   - 基于排名、评分、优惠、高频评论标签生成 1-2 句话
   - 示例："海底捞火锅(吴中路) 是上海最热门的火锅，24小时营业，评价4.9分，周末还有半价优惠"

3. **增强真实用户痛点**
   - 从评论中提取高频问题（排队、停车、儿童座位等）
   - 补充FAQ中的实际用户关切

### P1 优先级（重要）：
4. **优化热卖卡片布局**
   - 桌面端：排名 | 图片 | 信息（名称/评分/地址/优惠）
   - 移动端：竖向堆叠，分步展开

5. **添加商家后台 Dashboard**
   - 展示本月AI引用次数、排名、改进建议

6. **强化交互反馈**
   - Skeleton UI loading
   - Hover/Click 微动画
